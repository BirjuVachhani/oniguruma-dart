## 1.0.0

Initial release — Dart `dart:ffi` bindings to the native Oniguruma 6.9.10 regex
library, wrapped in a thin C shim (`src/oniguruma_shim.c`).

- One API on every platform (`OnigScanner`, `OnigString`, `OnigMatch`); the
  native engine is used on IO platforms, and web degrades gracefully
  (`isOnigurumaSupported` is `false`, construction throws).
- vscode-oniguruma-compatible multi-pattern scanning: `OnigScanner.findNextMatch`
  runs the whole scan loop in C, so there is exactly one FFI crossing per query.
- `OnigScanner.scanCount` counts every non-overlapping match across an input in a
  single FFI crossing (no per-match allocation).
- UTF-16LE encoding, so match offsets line up 1:1 with Dart `String` indices with
  no remapping.
- Native library provided by a Dart build hook (`hook/build.dart`): bundles a
  SHA-256-verified prebuilt for the target when one ships under `prebuilt/`,
  otherwise downloads the pinned Oniguruma source (SHA-256 verified) and compiles
  it with the shim. Opt into the source path with the `oniguruma.from_source`
  user-define.
- Prebuilt binaries for macOS, iOS (device + simulator), Linux, Android, and
  Windows; regenerated by the `prebuild-oniguruma` GitHub Actions workflow.

For a pure-Dart alternative that also runs on the web (and is faster for bulk
find-all-matches), see the sibling [`oniguruma_dart`](../oniguruma_dart) package.
